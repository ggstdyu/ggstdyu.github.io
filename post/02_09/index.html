被 Git 搅扰的第一天

Posted Feb 9, 2019

一起 Git 嗷~

关于 Git 精彩的布道

:-)

Git 是优雅的代名词，学习 Git 也要用优雅的方式 ，参考：
https://learngitbranching.js.org/

  1. Git Commit
    Git 仓库中的提交记录保存的是你的目录下所有文件的快照，它的提交记录非常轻量，它会将当前版本与仓库中的上一个版本进行对比，并把所有的差异打包到一起作为一个提交记录。Git 还保存了提交的历史记录，支持快速地在这些提交记录之间切换。
    git commit //提交
    :-)

  2. Git Branch
    Git 的分支也非常轻量，它们只是简单地指向某个提交纪录。早建分支！多用分支！即使创建再多分的支也不会造成储存或内存上的开销，并且按逻辑分解工作到不同的分支要比维护那些特别臃肿的分支简单多了。
    git branch <name> //创建name新分支
    git checkout <name> //指向name分支
    git checkout -b <name> //创建并指向name分支
    :-)

  3. Git Merge
    Git 合并两个分支时会产生一个特殊的提交记录，它有两个父节点。
    git merge <name> //合并name分支与当前指向分支
    :-)

  4. Git Rebase
    Git 支持第二种合并分支的方法，它的过程是取出一系列的提交记录，“复制”它们，然后在另外一个地方逐个的放下去，这样做的优势是可以创造更线性的提交历史。
    git rebase <name> //将当前支线所有提交记录移至name下
    :-)

  5. HEAD
    HEAD 是一个对当前检出记录的符号引用，也就是指向正在其基础进行工作的提交记录。HEAD 总是指向当前分支上最近一次提交记录。大多数修改提交树的 Git 命令都是从改变 HEAD 的指向开始的。分离的 HEAD 就是让其指向了某个具体的提交记录而不是分支名。
    :-)

  6. 提交移动
    Git 支持通过哈希值指定提交记录，但通过指定长达 40 位的提交记录哈希值在 Git 中移动不太方便，因此 Git 引入了相对引用。使用^向上移动一个提交记录，使用~<num>向上移动多个提交记录。
    git checkout <name>^ //指向name的父节点
    git checkout <name>~<num> //指向name的前num个节点
    git branch -f <name0> <name1>~<num> //强制将name0指向name1节点
    :-)

  7. 撤销变更
    Git 中撤销变更的方法有很多。Reset 用于本地分支，先前所做的变更还在，但是处于未加入暂存区状态。Revert 用于远程分支，要撤销的提交记录后面会添加一个新提交记录，用以撤销更改并推送到远程仓库与别人分享。
    git reset <name> //回退至name节点
    git revert HEAD //添加新变更以抵消
    :-)

  8. 整理提交记录
    Git 中整理提交记录的方法也有很多。Cherry-pick 将一些提交复制到当前所在的位置（HEAD）下面。交互式 Rebase 的参数简写为 -i，Git 会打开一个交互界面并列出将要被复制到目标分支的备选提交记录，它还会显示每个提交记录的哈希值和提交说明。
    git cherry-pick <name>...
    git rebase -i <name>
    :-)

几点问题：

  • Git与SVN的主要差别：版本历史库存在于每个本地，还是中央仓库；commit不需要网络，还是需要网络
git init
git add <file>
git commit -m <message>

git diff //比较暂存区与工作区（上次git add 的内容）
git diff HEAD //比较版本库与工作区（上次git commit 的内容）
git diff --staged //比较版本库与暂存区

git log //查看提交日志
git reflog //查看命令日志
git status //查看工作区状态
git log --graph --pretty=oneline --abbrev-commit //简单明了

git reset --soft <commit_id> //版本库回退到id的版本
git reset --mixed <commit_id> //版本库和暂存区回退(Unstaged changes after reset?)
git reset --hard <commit_id> //全部回退到id的版本
git checkout -- <file> //工作区回退到暂存区(或版本库???)
git reset HEAD <file> //暂存区回退到版本库(Unstaged changes after reset?)

git branch //列出分支
git branch <name> //创建分支
git checkout <name> //指向分支(工作区同时改变，惊了)
git branch -d <name> //删除分支
git checkout -b <name> //创建并指向分支
git merge <name> //合并指定分支到当前分支

关于 GitHub 精彩的布道

:-)

Git 是优雅的代名词，学习 GitHub 也要用优雅的方式。

  1. 注册 GitHub 账号
    :-)

  2. SSH 加密

    1. 创建 SSH Key
      终端键入ssh-keygen -t rsa -C "YourEmail@example.com"，密码设为无。
    2. 复制 SSH Key
      终端键入cat ~/.ssh/id_rsa.pub，复制公钥字符串。
    3. 添加 SSH Key
      登陆 GitHub，打开 Settings 页面，选择 SSH and GPG keys 选项卡，添加 SSH 公钥。
      :-)

  3. 第一个远程仓库

    1. 创建远程仓库
      Repository name 自定义，其他设置保持默认。
    2. 关联本地仓库
      打开终端，并进入仓库根目录。 git remote add origin git@github.com:GitHubName/RepoName.git
    3. 本地推送远程
      命令中的 -u 为关联本仓库与远程仓库的参数，再次推送时可以不添加该参数。
      git push -u origin master
    4. 克隆远程仓库
      若要将他人仓库下载到本地时，使用克隆命令，打开终端，并进入仓库根目录。 git clone git@github.com:GitHubName/RepoName.git :-)
