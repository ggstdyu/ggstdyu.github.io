被 Markdown 搅扰的第一天

Posted Feb 6, 2019

换了好看的皮肤嗷～

开启 Sublime Text

:-)

今天的任务是实现自动部署。因为 clone 下来的 py 程序有些冗余，我也不想调试，所以上午决定自己照着写一份精简版的。写完了准备测试，就 new 了 third.md，但编辑的时候又意识到没有好用的编辑器。知乎一番后，排除对于仅编辑文本而言太过臃肿的 Atom 和 VS Code，选择了 Sublime Text，但官方下载地址在墙外，速度极慢，几经周折找到附带注册码的安装程序。（悄悄说一声：支持正版！
https://pan.baidu.com/s/1kUTTC1p/

关于 Sublime Text 某些问题的解决方案：

  1. Sublime Text 安装插件。考虑到编辑的日志文本都是 Markdown 格式，所以安装几个相关插件：Package Control, Markown Editing, Markdown Preview, Live Reload, Markdown Live Preview。直接安装某些插件会提示下载失败，可以在 GitHub 上 clone，参考：
    https://blog.csdn.net/qq_20011607/article/details/81370236/

  2. Sublime Text 解决 Unable to download XXX 问题。一些插件似乎不能手动安装，但又是其他插件的基础部件：pygments, pymdownx, pyyaml, python-markdown。Sublime Text 会在后台自动安装它们，若未安装成功，则手动 clone 的一些插件不会正常运行，Sublime Text 会提示未能安装的插件，参考：
    https://www.jianshu.com/p/ebf3f39b37ca/

  3. Sublime Text 关闭自动更新。参考：
    https://jingyan.baidu.com/article/cdddd41c6ad82553ca00e14e.html

用 Sublime Text 简单编辑了本日志，测试自动部署程序，成功执行。
总之嘛，Python 简单起来还是很简单的。

还有许多想实现的功能，打算先拿相对基础的评论功能练练手，之后再把 GitHub 绿块块迁移过来，Contributions Calendar 冲鸭！
